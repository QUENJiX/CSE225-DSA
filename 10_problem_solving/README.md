# Problem Solving Workspace

Recommended layout:

- `solved/` - individual solved problems.
- `contests/` - timed contest files.
- `notes/` - reusable ideas, mistakes, and patterns.

Naming idea:

```text
topic_problem_name.cpp
```

Examples:

```text
binary_search_first_bad_version.cpp
graph_number_of_components.cpp
dp_climbing_stairs.cpp
```

Each problem file should eventually have:

- Problem link or source.
- Key idea.
- Complexity.
- Clean final solution.
